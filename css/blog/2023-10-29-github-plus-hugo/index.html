想節省租用個人網站主機費用?

沒想到有一天匯率升貶真的會有感影響生活，一直很希望可以經營自己網站，便買了域名還租用了cloudways主機並試著以wordpress架站，雖然wordpress免費，但需要有主機，想著財富自由後以後要靠經營網站生活，於是選擇付費租用主機，但用了大概半年後有兩大缺點，一是我真的不太會用wordpress導致套用主題後無法大幅度更動版面配置，二是cloudways主機在我申請的當下大概換算台幣一個月700多塊，最近因為匯率，換算台幣竟然一個月要台幣900多，有時又提不起勁更新文章，真的是好浪費啊！

感覺所有瀏覽器和app都在窺視我，怎麼我一覺得租用主機浪費，它就推播我廣告，我都還沒搜尋免費架站耶，難道它還會讀心術？！

總之，看到廣告跳出GitHub + HUGO免費架站線上課程，愛買課程的我當然二話不說就把卡刷下去，帶著滿腔熱血要來好好學一下，一心一意要讓每月可以省下近千元。

深入搜尋關鍵字發現網路上的文章共同提到“靜態網頁”這個詞，當時我連什麼是靜態網頁都不知道，跟著課程加上網路教學，沒想到我可以走到這裡。

推薦我上的這門課程，老師講得很清楚喔：Github 免費架站術！輕鬆打造個人品牌

下面就來筆記一下我看著課程還是卡關的地方

Github + HUGO 建立個人網站

簡單來說是套用HUGO主題，修改程式碼到滿意的樣子後，以HUGO產生程式碼，再用Github Desktop 建立一個commit發布到Github Page 上

說起來簡單但步驟就是一直來回修改，本機預覽，不滿意再改，再預覽，改到效果滿意後，產生程式碼，再上傳，再到網站看效果是否滿意，不滿意就再修，再傳。

其中我在上傳至Github後，發現頁面版型設定全部跑掉，圖片完全消失，似乎所有的html和css設定完全不見，只剩文字，詳情就詳下方說明吧

申請github帳號創建repository

這個步驟就照著github 網站註冊流程註冊一個新帳號，每個github 帳號都有一個github page可以使用，預設網址就會是https://yourgithubid.github.io，用來放個人頁面的repository 必須以yourgithubid.github.io。

(yourgithubid是指你自己申請的github帳號)

這個步驟想當順利，唯一有點後悔的是，因為常用帳號已經被註冊，導致我加了後面幾碼，但我不知道哪根筋不對勁，竟然加了大寫在後面，目前為止，大小寫問題對網頁建立沒有造成影響，但還是建議全小寫比較好。

挑選並套用HUGO主題

先下載電腦相應的hugo程式解壓縮備用，然後挑選喜歡的主題下載套用。

要憑空從無到有架出一個可以看的網站不容易，是連code和html都不懂的初學者也沒關係，只要到HUGO上的Themes從中挑出喜歡的主題，或是離你心中理想的頁面分布比較相近的主題，下載程式碼後再修改成你想要的網站設定後上傳就可以。

將下載下來的主題解壓縮，找到example site資料夾複製到桌面，並在資料夾中新增一個資料夾，命名為themes，然後把下載下來解壓縮後的資料夾整個移到或複製一份到剛剛新增的themes資料夾中。

接著把這個步驟剛開始下載解壓縮備用的hugo程式複製一份到example site資料夾，開啟命令提示字元，鍵入hugo server運行網頁，把產生的網站貼到瀏覽器上就可以在本機電腦預覽網頁效果了。

安裝VS code更改HUGO主題設定

接下來安裝一個可以編輯code的vs code開啟example site資料夾，從主頁面config.toml到各個頁面(比如有的主題有portpolio,blog,contact…)去修改你想要網站呈現的字樣，更換你想顯示的圖片後存檔，在到瀏覽器預覽修改效果，邊改邊存檔。

將hugo產生的程式碼上傳至Github

好了，我要來說以Github + HUGO 架站的過程中最令我崩潰的事，也就是文章開頭提到慘案，事情是這樣的，在我將修改好基本首頁設定，將hugo產生的程式碼上傳至Github後，發現頁面版型設定全部跑掉，圖片完全消失，似乎所有的html和css設定完全不見，只剩文字，what ?!!!!

我當然立刻請問了Google 老師，檢查了幾個可能的錯誤，包括圖片路徑設定是否正確(雖然排除了這個問題，但我後來發現自己打路徑真的有夠容易打錯，尤其是在rename資料夾名稱後，常常因為沒更新路徑，讓圖片消失)，github 帳號和設定的baseURL是否正確，確定是githubid.github.io也沒錯，一一嘗試搜尋到的解方都宣告失敗。

不行，真的不行。

後來想到會不會是github 或 hugo對大小寫敏感，我的github 帳號中同時有大小寫的關係，為了測試，我又重新申請了一個github 帳號，全部小寫，然後把上述流程又給重操了一遍，滿心期待我是不是要解開了，結果再度宣告失敗。

不行，怎樣都不行。

再次求助Google 老師，發現大家的解方都落在root和URL必須一致居多，我只好再度回頭把repository name 和baseURL又再看一遍，剛剛有提到圖片路徑用打的很容易打錯吧，我心想核對了一遍又一遍都沒錯的網址難道有肉眼看不見，但電腦認為的不一致嗎？

我保持這最後希望，把版型跑掉、看起來比純文字更不堪的Github Page網址複製，貼到config.toml的baseURL中，產生程式碼，再上傳一次試試，結果你知道怎麼著，Github Desktop 偵測我改動的地方是http"s"://，s，就是這個s，讓我的所有設定都回來了啊啊啊啊啊！！！！

各位，別像我一樣犯了個愚蠢的失誤。

Read more位置設置

設定好首頁、個人介紹頁面後，終於來到Blog文章搬家的時刻，本來以為就是一篇篇複製貼上，雖然有點麻煩但應該沒什麼難度，但這個想法在我貼完兩篇文章後就終結了。

原本套用的版型會顯示一張圖片，加上文章的的前幾句，但不知道為什麼，我貼上我的文章後他就是整篇文字都顯示，超出了文章區塊的設定。

看了一下程式碼，明明有設定顯示前70個字就接…read more但就是起不了作用，後來嘗試了兩種方法解決這個問題，第一種比較笨，也比較沒效率，就在文章想顯示read more 的地方插入html語法

就可以了，好處是你想顯示多少字，愛怎麼切都隨你，但…每篇文章都得手動貼上語法。

另一種就是直接修改程式碼，讓blog首頁呈現一句文章介紹就好，在每篇文章新增一段description : ，然後指定首頁呈現敘述的內容就好。

文章搬家

經歷了上面的波折，總算可以到真的只剩文章搬家的工作了，雖然文章內容搬家實在是有點無腦複製貼上，但也算簡單，一篇篇把文章新增.md檔到blog資料夾下就大功告成了，不知道有沒有批次搬運這種程式，感覺這種重複性高的工作，應該可以讓程式去完成。

不過初學者只能土法煉鋼，一篇一篇複製貼上，但好處是再搬運的過程，可以再看一下文章，也能再調整一下內容或之前在wordpress 主題不知道怎麼更改的版面配置。

網站搬家一周後心得

要用Github+HUGO 的方式寫blog好像就沒辦法用手機隨時隨地發文(？)感覺必須要用筆電才能上傳整個site的資料夾，還說可以直接上傳文章的.md檔就好？再試個一陣子看看，付費有後台可以編輯就是比較友善，但可以達成省下千元的目標也是值得了。

本次網站搬家作業到這個階段暫告一個段落，

筆記部分後續還想補「綁定GitHub Pages」 和「 Namecheap 域名 DNS 設定」、「設定Disqus」，「設定Google Analytics 4 分析網站流量」及「上傳Site map 被搜尋功能」

網站內容部分，還想把文章目錄的部份調整美化一下，再把Gallery頁面內容新增上去

不過在職生期中考將至，我得先去抱下佛腳…只能先擱著了。
